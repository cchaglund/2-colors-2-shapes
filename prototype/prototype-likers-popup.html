CHOSEN: Variant D — Inverted Tooltip (Above). The other variants (A, B, C) are archived below for reference.

UI/UX Decisions & Notes

Trigger & positioning: The tooltip appears on hover of the CardLikeButton (size="lg") on SubmissionDetailPage. It must be centered on the button's full width (including the expanded "Like"/"Unlike" text that appears on hover). The like button's existing hover animation (count slides out, label slides in) fires simultaneously with the tooltip appearing. Use @floating-ui/react (already used in InfoTooltip.tsx) with flip() to auto-reposition when near viewport edges, and safePolygon() for reliable hover bridging.

Tooltip styling: Dark inverted colors (--color-text-primary bg, --color-bg-primary text) — matches the existing Tooltip in InfoTooltip.tsx. Each row: small avatar (22px, use AvatarImage component) + nickname. Arrow points down, centered on the button. Rows are clickable (navigate to user profile).

Content rules: Show at most ~5 likers. If there are more, show a "View all N likes" footer link. Clicking that footer opens the Likers Modal (see prototype/prototype-likers-modal.html). The footer has a subtle separator line and underlines on hover.

Companion modal: The "View all N likes" click opens a full likers modal — see prototype/prototype-likers-modal.html for that design. It shows ALL likers with avatars, nicknames, and Follow buttons (only for non-followed users).

A — Classic Dropdown List

Vertical list anchored below the button. Each row shows avatar + nickname. Header label, scrollable body, "+ N more" footer.

Why: Most scannable layout for names — users identify specific people fastest in a vertical list. The header ("Liked by") gives context. Capped height with scroll prevents the popup from dominating the page. Most accessible: screen readers traverse linearly, keyboard nav works. Row hover gives clear affordance each user is clickable.

Interaction note: The like button expands on hover (count slides out, "Like" slides in). This popup appears below, so the button's shape change doesn't conflict — the popup just shifts down slightly as the button widens.

B — Facepile with Expanding Dropdown

Overlapping avatar circles sit persistently next to the like button. Hovering the facepile reveals a named list.

Why: Faces are always visible as social proof — psychologically more motivating than a bare number. The overlapping layout is compact and distinctive. This is the only prototype where likers are partially visible without hovering.

Interaction note: Here the popup is on the facepile, not the like button itself, so the button's hover expansion doesn't interfere. The button and facepile sit side-by-side — the button does its count-to-label thing independently while the facepile has its own hover dropdown.

C — Avatar Grid Card

Wider floating card with avatars in a flowing grid, names beneath each, accent-colored header with heart icon.

Why: Emphasizes faces over text — fits a visual/creative community. The 2D grid scales better than a 1D list for high like counts. Avatar hover rings add micro-delight. Feels like content discovery rather than a utility dropdown.

Interaction note: Appears below. The button's expansion is small enough (~20px wider) that the popup doesn't need to reposition. However, the wider card format means the popup extends well beyond the button width — it anchors to the left edge.

D — Inverted Tooltip (Above)

Compact dark tooltip appearing above the button with small avatars and names. Matches the existing Tooltip component's inverted color scheme.

Why: Lightest-weight option and most consistent with existing patterns (InfoTooltip uses dark-bg/light-text inverted scheme). Appearing above avoids obscuring the canvas artwork below the like button. The inverted colors create clear visual separation from page content. Best when the liker list is supplementary info.

Interaction consideration: The button's hover expansion (count slides up and out, "Like"/"Unlike" slides in) happens simultaneously with the tooltip appearing above. Because the tooltip is above and the button expands horizontally, there's no visual collision — they complement each other. The arrow is left-aligned to point at the heart icon.

"+N more" behavior: The footer is a clickable link (hover to see underline). Options for what it does: (a) Scroll the page to a dedicated "Likes" section below the artwork (if one existed). (b) Open a small modal/drawer listing all likers with follow buttons — this is what Instagram does. (c) Simply show more rows inline in the tooltip (lazy-load the next batch). This keeps the user in context but risks the tooltip growing too tall. Recommendation: (b) — a modal is the cleanest separation between "quick peek" (tooltip) and "full list" (modal with scroll + follow buttons).

Production note: Use @floating-ui/react with flip() + offset(8) + shift(). The flip() middleware handles the case where the button is near the top of the viewport — the tooltip will automatically appear below instead. The invisible bridge element won't be needed in production because floating-ui's useHover has a built-in handleClose option (safePolygon) that keeps the popup open while the cursor moves between trigger and popup.
